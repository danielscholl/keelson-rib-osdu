The design tier is where the reasoning lives: decision records, and the
Keelson base gaps this rib was built against.

The rib's build was sequenced by five gaps in the Keelson base, `G0` through
`G4`: table cell tone, the composite `board` view kind, card link and copy
affordances, the rib action round-trip, and the top-level surface with its
region layout. Each was domain-free and reusable by any rib, which is the
point: the rib asked the harness for primitives, not for OSDU knowledge. All
five have shipped, and the analysis that sequenced them is kept as a
historical record in
[`docs/design/base-gaps-history.md`](https://github.com/danielscholl/keelson-rib-osdu/blob/main/docs/design/base-gaps-history.md).

Pages land here as the documentation grows.
