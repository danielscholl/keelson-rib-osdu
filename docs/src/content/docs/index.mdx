Operating an OSDU CIMPL cluster means juggling five CLIs: `cimpl` for the
cluster, `kubectl` for Flux, `osdu-quality` for release health,
`osdu-activity` for delivery motion, and `glab` for GitLab. Each answers one
question; none shows the whole platform at once.

This rib turns that toolchain into **one live surface** inside
[Keelson](https://danielscholl.github.io/keelson/): cluster health and
lifecycle, the Flux dependency graph, and platform-health lanes for quality,
features, security, the release train, your personal queue, and recent
events. Every panel is fed by a deterministic workflow whose collector shells
one of those CLIs and shapes the JSON with a pure, tested builder.

Three design choices keep it honest:

- **No sidecar.** Every view is gathered through one-shot CLI invocations;
  nothing stays resident.
- **No domain logic in Keelson.** OSDU and CIMPL knowledge lives in the rib;
  the harness stays domain-free.
- **No trusted React from the rib.** Views render through Keelson's canvas
  primitives (`board` and `graph`), never hand-coded UI.

`@keelson/rib-osdu` is a Keelson **rib**: a discovery-based extension that
attaches through the `Rib` contract from `@keelson/shared`. Keelson's own docs
cover the generic contract; this site documents the one concrete rib that
implements it.
